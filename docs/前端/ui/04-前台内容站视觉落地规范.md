# 前台内容站视觉落地规范

## 文档目标

本文档用于定义 `Yunyu` 前台内容站的共用视觉规范，确保首页、文章页、分类页、专题页、标签页在继续升级时，始终落在同一套品牌语言与版式规则中。

本文档讨论的是前台页面的长期共用规范，包括：

- 品牌标题系统
- 前台页面栅格与留白
- 内容块与卡片的使用边界
- 图片与封面的统一规则
- 页面动效原则
- 前台语义化视觉 token 的落地约束

本文档不负责记录单一页面的临时实现细节。

## 一、前台视觉总纲

`Yunyu` 前台页面的统一目标不是“每页都好看”，而是“整个站点看起来像同一个内容品牌”。

因此前台页面统一遵循以下原则：

- 内容优先于组件表现
- 品牌优先于组件库默认样式
- 版式优先于卡片数量
- 留白优先于装饰堆叠
- 明亮模式优先，暗黑模式克制

前台整体视觉气质保持为：

- Editorial
- Minimal
- Calm
- Atmospheric
- Structured

## 二、品牌标题系统

### 2.1 字体组合

前台统一使用不超过两类主字体：

- 标题字体：`Playfair Display`
- 正文字体：`Inter`
- 中文回退字体：`PingFang SC`、`Microsoft YaHei`

使用原则如下：

- 品牌名、Hero 标题、文章大标题、专题大标题优先使用标题字体
- 正文、导航、说明文字、筛选项、辅助信息统一使用正文字体
- 不允许前台页面自由引入第三类展示字体

### 2.2 标题层级

前台标题层级统一分为五档：

- 品牌标题：用于首页或重要入口的品牌识别
- 页面 Hero 标题：用于首页、文章页、分类页、专题页的大标题
- 分区标题：用于页面内部大区块
- 内容标题：用于文章卡片、专题条目、分类条目
- 辅助标题：用于目录、侧栏、小节说明

统一要求如下：

- 品牌标题必须是页面最有识别度的文本
- 页面 Hero 标题必须在首屏一眼可读
- 分区标题不能与内容标题混成同一级
- 同一页面中，标题层级不能频繁跳变

### 2.3 文案密度

前台标题和副文案必须克制：

- 首屏标题控制在可一眼扫完的长度
- 副文案只做补充，不重复标题含义
- 页面中不允许大段“设计说明式”文案
- 标签、徽标、角标只能起辅助作用，不能淹没主标题

## 三、布局与留白规范

### 3.1 版式优先级

前台页面布局顺序统一为：

1. 先确定首屏主视觉
2. 再确定正文或内容流的阅读宽度
3. 再确定分区之间的呼吸感
4. 最后才是组件外观细节

### 3.2 页面容器

前台页面统一保持以下约束：

- Hero 区优先全幅或近全幅表达
- 页面正文与内容流使用稳定的最大宽度
- 不同页面的主内容区宽度变化不能过于跳跃
- 移动端优先保证标题与正文的呼吸感，而不是尽量塞更多内容

### 3.3 留白规则

前台页面的高级感主要依赖留白而不是阴影，因此统一要求：

- 分区之间必须有明显大于组件内部间距的区隔
- 内容块内部间距稳定，不要每块都不同
- 页面首屏到底部内容之间要有自然过渡，不可突兀切断
- 长内容页中，章节切换留白必须明显强于普通段落留白

## 四、内容块与卡片使用边界

### 4.1 总原则

前台不是后台工作台，不允许默认把所有内容都做成白底卡片。

因此统一要求：

- 卡片只在需要明确承载一个独立内容对象时使用
- 同一区域内不应堆满样式完全一致的卡片
- 页面优先通过图文比例、留白、分栏、细分隔建立层级

### 4.2 可使用卡片的场景

以下场景可以使用卡片化表达：

- 首页推荐阅读中的独立文章对象
- 分类或专题中的重点入口
- 需要承载封面图、标题、摘要和元信息的一体化内容单元

### 4.3 应避免过度卡片化的场景

以下场景应优先使用非卡片化版式：

- 首页的主打内容导流区域
- 内容路径说明区
- 多条内容的连续阅读列表
- 分类地图、专题路线等更偏导览性质的区域

### 4.4 内容块层级

前台内容块至少应区分以下三类：

- 主打内容块：负责吸引进入
- 普通内容块：负责稳定承载文章或专题
- 导览内容块：负责告诉用户如何继续阅读

后续组件实现时，不能只依赖一个通用卡片组件覆盖全部场景。

## 五、图片与封面规则

### 5.1 图片角色

前台图片不能只是装饰，必须承担叙事作用。

统一要求如下：

- 首页首屏图片必须具备主视觉承担能力
- 列表图与封面图必须服务于内容气质，而不是只满足“有图可放”
- 图片上承载文字时，必须保证可读性和稳定对比

### 5.2 裁切与比例

统一要求如下：

- 同一列表中的图片比例保持稳定
- 主打内容图与普通条目图必须有清晰层级差异
- 图片裁切优先保留氛围主体，不为了对齐强行裁掉关键区域
- 移动端需要单独验证图片裁切后的文字安全区

### 5.3 图片风格

前台图片风格统一倾向：

- 情绪明确
- 色调稳定
- 细节不脏乱
- 避免内嵌杂乱文字或标识干扰页面排版

## 六、动效与交互规范

### 6.1 动效目标

前台动效只服务三个目标：

- 建立首屏存在感
- 强化分区层级
- 提升 hover 与点击的轻反馈质量

### 6.2 推荐动效

当前前台页面统一优先使用以下三类动效：

- 首屏标题与副文案的分层渐显
- 内容分区进入时的弱位移与透明度过渡
- 图片 hover 的轻缩放与明暗变化

### 6.3 动效边界

统一禁止以下情况：

- 大幅度弹跳和夸张缩放
- 多个区域同时做强动效竞争注意力
- 纯炫技而不提升阅读体验的复杂动画
- 暗黑模式下使用过亮的发光 hover 效果

## 七、前台视觉 token 约束

### 7.1 使用方式

前台页面后续必须逐步从散写颜色类收口为语义 token。

优先约定以下语义层：

- `bg-page`
- `bg-surface`
- `bg-surface-soft`
- `text-strong`
- `text-body`
- `text-muted`
- `border-default`
- `primary`

### 7.2 落地要求

统一要求如下：

- 新增前台页面样式时，优先使用语义 token，而不是直接写具体颜色
- 旧页面在重构时逐步收口，不要求一次性全部替换
- 同一类 surface 的亮暗主题表现必须保持对应关系
- 品牌强调色只承担强调作用，不能铺满整页

## 八、页面类型落地要求

### 8.1 首页

首页必须满足：

- 首屏品牌感最强
- 主打内容最醒目
- 推荐内容、分类地图、专题路线、最新内容之间有明确节奏

### 8.2 文章页

文章页必须满足：

- 首屏有封面感
- 正文是页面主角
- 目录、延伸阅读、标签只做阅读辅助

### 8.3 分类页、专题页、标签页

入口页必须满足：

- 有自己的头图或头部表达
- 不只是筛选结果列表
- 能告诉用户“从这里怎么继续逛”

## 九、文档与实现协同规则

前台视觉相关能力后续统一按以下方式推进：

1. 先在文档中明确共用规则
2. 再做组件抽象或页面改造
3. 改造后若出现新的稳定规则，继续补回文档

如果某个实现只适用于某一个页面，则不应直接上升为共用规范。

## 十、当前正式结论

当前项目前台视觉落地正式约定如下：

- 前台页面统一遵循“品牌优先、版式优先、留白优先、内容优先”的设计原则
- 前台标题系统、图片规则、卡片使用边界、动效边界和视觉 token 必须长期统一
- 后续首页、文章页、分类页、专题页、标签页升级时，均应以本文档作为共用规范依据
