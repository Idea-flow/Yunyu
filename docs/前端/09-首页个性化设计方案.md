# 首页个性化设计方案

## 文档目标

本文档用于定义 `Yunyu` 首页从“已经具备内容聚合能力的首页”升级为“有强记忆点、有个人气质、有沉浸阅读氛围的最美博客首页”的设计方案。

本文档重点回答以下问题：

- 首页应该走什么视觉路线
- 首页是否需要 `3D`
- 首页背景是否需要做得更有特点
- 首页是否需要单独配置
- 首页是否需要增加一份 `MySQL` 配置表或首页专属配置结构

本文档是后续首页重构、后台配置扩展、前后端接口调整的共同依据。

## 一、当前首页判断

结合当前实现，首页已经具备以下基础：

- 已有较完整的首页内容结构，包括 Hero、推荐阅读、分类地图、专题路线、最新内容
- 已有较明确的前台视觉方向，偏向 `Editorial`、`Atmospheric`、`Magazine`
- 已有基础站点配置能力，可通过 `site_config` 承载部分首页风格字段

但当前首页距离“最美博客网站 / 最美内容网站”仍有以下差距：

- 品牌记忆点还不够强，首屏更像“漂亮首页”，还不像“独属于云屿的首页”
- 首页背景虽然已有渐变和氛围层，但辨识度还不够高
- 模块之间的节奏已经存在，但“首页专属叙事”还不够集中
- 首页配置仍偏基础站点字段，缺少真正为首页服务的独立配置模型

## 二、首页设计总命题

`Yunyu` 首页不应做成“模板博客”，而应做成：

- 一本有封面感的数字杂志首页
- 一个可以慢慢逛的内容入口页
- 一个带有轻叙事、轻氛围、轻沉浸的个人品牌首页

首页的视觉关键词统一为：

- `Editorial`
- `Atmospheric`
- `Immersive`
- `Calm`
- `Curated`
- `Narrative`

一句话定义：

> 首页不是展示“我发了哪些文章”，而是告诉用户“应该从什么情绪和路径进入我的内容世界”。

## 三、风格决策

### 3.1 推荐主风格

首页推荐采用以下组合路线：

- 主风格：`Editorial Grid / Magazine`
- 氛围层：`Atmospheric Gradient`
- 结构语言：`Storytelling-Driven`
- 信息组织：`Content First`

这套组合比纯 `Bento`、纯 `SaaS`、纯 `Glass` 更适合内容站，也更适合长期阅读型博客。

### 3.2 不建议的方向

首页不建议走以下方向：

- 纯后台式数据面板风
- 过度卡片堆积
- 赛博霓虹重特效风
- 整屏大面积液态玻璃
- 大量 `WebGL` / 粒子 / 强滚动视差的炫技型体验

原因很明确：

- 它会抢走内容本身的注意力
- 会拉低阅读站点应有的克制感
- 会增加移动端性能和维护成本

## 四、首页是否需要 3D

### 4.1 结论

首页**不建议做重型 3D**，但**建议做轻量 2.5D 空间层次**。

### 4.2 为什么不建议重型 3D

对于博客和内容站，重型 `3D` 常见问题是：

- 首屏资源过重
- 与文章阅读场景冲突
- 容易从“高级”做成“花哨”
- 后续每次换内容都要兼容复杂视觉资产

### 4.3 推荐替代方案

更适合 `Yunyu` 的不是真正的 `3D`，而是以下“拟 3D”表达：

- 分层背景：远景氛围光、主体背景图、前景漂浮信息卡
- 卡片叠层：主打文章卡轻微压在背景封面之上
- 光影纵深：通过模糊、透明度、柔和投影制造空间感
- 微透视插画：如果后续有插画资源，可使用轻透视构图，但不做实时交互 3D

最终目标是让首页“有深度”，而不是“像一个 3D 演示页”。

## 五、首页背景是否需要有特点

### 5.1 结论

首页背景**必须有特点**，但这个特点应该来自“品牌气质”，不是来自“复杂效果”。

### 5.2 推荐背景策略

首页背景建议采用“四层结构”：

1. 底层纸感渐变
2. 中层情绪光斑
3. 上层云雾 / 星空 / 颗粒纹理
4. 局部品牌图形

适合 `Yunyu` 的背景语义建议：

- 云海
- 岛屿
- 夜空
- 风的轨迹
- 轻微星点
- 漂浮的阅读坐标

### 5.3 推荐实现方式

建议背景以 `CSS gradient + noise texture + soft radial glow + mask` 为主：

- 明亮模式：纸张感、云雾感、浅天光
- 暗黑模式：深海蓝、夜空蓝、星屿感
- 局部点缀：使用少量品牌橙或天青做发光引导

### 5.4 不建议的背景

不建议以下背景方式：

- 大面积视频背景
- 高频动态粒子
- 过亮的流体渐变
- 纹理过重导致文字可读性下降

## 六、首页信息架构建议

首页建议统一为六段式结构。

### 6.1 第一屏：品牌封面 Hero

作用：

- 建立第一眼记忆点
- 强化“云屿”品牌感
- 给出一条清晰进入路径

建议内容：

- 品牌眉题
- 一句强主张标题
- 一段较短副文案
- 主打文章入口
- 一个主 CTA
- 一个次 CTA
- 站点摘要统计

建议视觉：

- 全幅或近全幅背景图
- 主打内容卡叠加在背景之上
- 标题与摘要分层渐显
- 右侧或下方出现“本周主打文章”

### 6.2 第二屏：编辑精选

作用：

- 承接首屏后第一波点击
- 体现站点品味，而不是只展示最新

建议内容：

- 3 篇精选文章
- 每篇有封面、标题、摘要、分类 / 专题标签

建议视觉：

- 不做完全相同的三张白卡
- 采用不对称杂志栅格
- 其中 1 篇可以更大，形成视觉主次

### 6.3 第三屏：阅读地图

作用：

- 告诉用户“从哪里开始逛”
- 降低首页只看标题卡片的疲劳感

建议内容：

- 分类地图
- 专题路线
- 阅读方向说明

建议视觉：

- 分类像“入口地图”
- 专题像“连续阅读路线”
- 配一个简短的策展式说明块

### 6.4 第四屏：站长手记 / 本站序言

作用：

- 增加个人气质
- 让首页不只是文章导流页，而是有人的存在感

建议内容：

- 一段 80 到 140 字的站长短文
- 一张站长头像或代表性图片
- 一个“关于我 / 关于本站”入口

这个模块非常重要，它往往是“博客像博客”的关键。

### 6.5 第五屏：最新内容流

作用：

- 维持更新感
- 给用户“继续逛下去”的动力

建议内容：

- 1 条最新主打
- 4 到 6 条最新文章流

建议视觉：

- 采用内容流，不要每条都做成重卡片
- 重点是阅读节奏、行距、分隔和图片比例

### 6.6 第六屏：订阅 / 归档 / 结束 CTA

作用：

- 为首页收尾
- 给用户下一步动作

可选内容：

- 邮件订阅
- 文章归档入口
- 主题导航入口
- 关于页面入口

## 七、首页需要什么“个性化设计”

真正能让首页有个性的，不是多加几个动效，而是首页有一套专属表达。

推荐增加以下个性化内容：

- 首页主标题支持手动配置，不强依赖当前推荐文章标题
- 首页副标题支持按季度或主题切换
- 首页 Hero 支持单独背景图 / 背景模式
- 首页支持“本周主打语”或“策展语”
- 首页支持“站长手记”
- 首页支持“阅读路径说明”
- 首页支持“技术栈 / 内容标签 / 站点情报卡”开关

## 八、是否需要展示 MySQL 表、配置信息、技术卡片

### 8.1 结论

可以加，但**不能做成首页主角**。

### 8.2 推荐做法

如果 `Yunyu` 目标是偏技术博客、开发内容站，可以在首页中后段加入一个轻量“站点情报卡”模块，展示：

- `Nuxt`
- `Spring Boot`
- `MySQL`
- `Markdown`
- 内容数 / 分类数 / 专题数

它应该更像“站点小档案”或“内容工程卡”，而不是后台报表。

### 8.3 不推荐做法

不建议在首页直接放原始数据库表格、真实表结构图、复杂 SQL 面板。

原因：

- 会破坏首页的内容氛围
- 容易让首页从“内容品牌站”变成“技术展示页”

更好的方式是：

- 首页只展示简洁技术卡
- 详细数据库设计放到专题文章页或“关于本站”页面

## 九、首页专属配置方案

在现有 `site_config` 中新增一条配置：

- `config_key = homepage_config`

## 十、`MySQL` 设计建议

### 10.1 第一优先方案

第一阶段建议继续使用现有 `site_config` 表，不新增首页总表。

原因：

- 当前项目已经具备配置中心雏形
- 首页配置本质上属于站点配置的一部分
- 可以更快落地，避免过早拆表

### 10.2 需要新增表的场景

如果后续首页需要支持以下能力，再考虑单独建表：

- 首页模块拖拽排序
- 不同模块独立上下线
- 同一模块多版本内容
- 首页多套主题切换
- 更复杂的运营位管理

### 10.3 可选表结构

如果未来要扩展首页运营能力，建议新增：

```sql
CREATE TABLE site_homepage_module (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键',
  module_key VARCHAR(64) NOT NULL COMMENT '模块唯一键',
  module_name VARCHAR(128) NOT NULL COMMENT '模块名称',
  enabled TINYINT(1) NOT NULL DEFAULT 1 COMMENT '是否启用',
  sort_no INT NOT NULL DEFAULT 0 COMMENT '排序号',
  config_json JSON NOT NULL COMMENT '模块配置 JSON',
  remark VARCHAR(255) DEFAULT '' COMMENT '备注',
  created_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  UNIQUE KEY uk_module_key (module_key)
) COMMENT='首页模块配置表';
```

推荐模块键：

- `hero`
- `featured`
- `reading_map`
- `editorial_note`
- `latest_stream`
- `tech_card`
- `footer_cta`

## 十一、首页设计令牌建议

首页建议新增专属视觉 token，而不是继续全部写死在页面类名中。

建议增加：

- `--home-bg`
- `--home-bg-soft`
- `--home-glow-primary`
- `--home-glow-secondary`
- `--home-hero-overlay`
- `--home-card-radius-xl`
- `--home-section-gap`
- `--home-text-hero`
- `--home-text-muted`

## 十二、最终设计决策

### 12.1 关于 3D

- 不做重型 `3D`
- 采用轻量 2.5D 分层
- 用光影、模糊、叠层代替真实三维交互

### 12.2 关于背景

- 背景必须有特点
- 特点来自“云、屿、夜空、阅读坐标”的品牌意象
- 背景要克制，不能压过标题和内容

### 12.3 关于首页内容

- 首页应该更像数字杂志封面
- 不应继续堆更多同质卡片
- 应加入“站长手记”和“阅读路径说明”来增强个人气质

### 12.4 关于配置与数据库

- 第一阶段复用 `site_config`
- 增加 `homepage_config`
- 第二阶段若首页运营能力增强，再拆 `site_homepage_module`

## 十三、推荐落地顺序

### P0

- 确定首页视觉主题
- 确定 Hero 新文案和背景策略
- 增加首页专属配置结构

### P1

- 重构首页 Hero
- 增加站长手记模块
- 重做阅读地图与专题路线区

### P2

- 增加技术情报卡可选模块
- 收口首页 token
- 后台增加首页配置编辑项

### P3

- 若确实有运营需求，再新增首页模块表
- 继续补动画、移动端与暗黑模式细节

## 十四、结论

`Yunyu` 首页想做到“最美博客网站”，关键不在于是否上 `3D`，而在于是否建立了独属于这个站点的品牌封面感、背景气质、首页叙事和配置能力。

因此本项目首页下一阶段的正确方向应该是：

- 用杂志化版式建立高级感
- 用氛围背景建立识别度
- 用轻量空间层次建立沉浸感
- 用首页专属配置建立可持续演进能力

后续如果进入实际开发阶段，应先从 `Hero + 首页配置 + 站长手记` 三部分开始落地。
