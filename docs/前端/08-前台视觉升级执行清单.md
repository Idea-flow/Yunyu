# 前台视觉升级执行清单

## 文档目标

本文档用于把 `Yunyu` 前台从“当前已具备杂志化雏形的内容站”继续升级到“更强品牌感、更强阅读沉浸感、更统一视觉语言”的具体执行项。

本文档关注的是：

- 下一步先做什么
- 每一项为什么重要
- 每一项要交付到什么程度
- 哪些内容需要同步到共用规范中

本文档不是设计灵感收集，也不是长期愿景说明，而是前台视觉升级阶段的执行清单。

## 一、当前阶段判断

结合当前前台实现与现有文档，`Yunyu` 已经具备以下基础：

- 首页已经建立大图 `Hero` 与杂志式内容导流结构
- 文章详情页已经具备大图首屏、目录联动、延伸阅读等基础能力
- 分类、专题、标签等入口页已经成型
- 已开始沉淀 `YunyuHero`、`YunyuSectionTitle`、`YunyuImage` 等公共视觉组件

当前距离“最美博客网站 / 最美内容网站”的主要差距，不在于页面数量不够，而在于以下几个层面还不够统一和极致：

- 品牌标题系统还不够鲜明
- 列表区与入口区仍有明显卡片站痕迹
- 正文排版节奏仍可继续精修
- 整站视觉 token 还没有完全收口
- 页面之间虽然方向一致，但还缺少更强的一致性约束

## 二、本阶段升级目标

本阶段前台视觉升级只聚焦四个目标：

1. 建立更强的前台品牌气质
2. 降低“卡片堆叠感”，提升内容编排感
3. 提升文章详情页的沉浸式阅读体验
4. 把前台视觉规则正式沉淀为共用规范

## 三、执行优先级

### P0：前台品牌系统升级

这是当前最优先的部分，也是最直接影响“第一眼是否高级”的部分。

#### 目标

- 让用户第一眼记住的是“云屿”，而不是“一个做得不错的博客模板”
- 让首页、专题页、文章页拥有统一的品牌标题语气
- 让前台页面形成更稳定的视觉母题

#### 执行项

- 统一首页、文章页、专题页、分类页的大标题语气与字号层级
- 明确前台标题字体、正文字体、中文回退字体的正式组合
- 为品牌标题、Hero 标题、分区标题、文章标题建立固定层级
- 收敛首屏标签、徽标、弱信息数量，避免首屏信息过满
- 明确前台统一视觉母题，例如“纸感、海雾、夜空、岛屿、星点”等氛围元素的使用边界

#### 交付标准

- 首页首屏品牌信息更强，副信息更克制
- 页面级标题在多个页面之间具有统一识别度
- 不同前台页面看起来像同一个内容品牌，而不是同一技术栈下的多个页面

### P1：内容编排升级

这一阶段的目标，是让前台从“漂亮组件组合”继续升级成“有编辑感的内容站”。

#### 目标

- 降低通用卡片感
- 强化图文比例与留白带来的高级感
- 让每个页面先有版式主次，再有组件细节

#### 执行项

- 首页推荐阅读区至少保留一种“非标准卡片化”的主打内容版式
- 分类页、专题页、标签页区分“主打条目”和“普通条目”的版式
- 弱化统一大圆角白底卡片的依赖，更多使用留白、细分隔、图文比例建立层级
- 控制同一屏中相似组件的重复度，避免整个页面都是同一张卡片的平铺
- 重新审视 `FrontPostCard` 的职责，拆分为更清晰的几种前台内容块版式

#### 交付标准

- 首页、分类页、专题页在结构上更有“内容编排感”
- 同页内容块具备主次差异，不再全部长得相似
- 前台整体更像数字杂志，而不是卡片列表站

### P2：文章详情页精修

文章详情页是内容站最能拉开差距的页面，因此本阶段需要继续做精修，而不是只停留在“功能完整”。

#### 目标

- 进一步强化长文阅读舒适度
- 提升文章视觉节奏与章节呼吸感
- 让目录、引用、图片、代码块成为阅读体验的辅助，而不是阅读噪声

#### 执行项

- 重新校准正文宽度、字号、行高、段间距和章节间距
- 对 `h2`、`h3`、段落、列表、图片、引用、代码块建立更清晰的上下间距规范
- 优化正文插图与图片说明区的留白关系
- 控制目录侧栏权重，避免在桌面端与正文争抢注意力
- 把相关文章区从“更多文章”升级为“沿当前主题继续阅读”

#### 交付标准

- 正文阅读节奏明显比当前更舒展
- 标题与正文的呼吸感更自然
- 文章页具备更强的沉浸式阅读体验

### P3：前台视觉 token 收口

这一阶段的目标，是让设计语言真正稳定下来，避免页面继续增长后视觉再次发散。

#### 目标

- 把高频出现的前台颜色、背景、边框、文字层级收敛成语义 token
- 统一明亮模式与暗黑模式下的页面表面层级
- 降低页面里到处分散写样式的维护成本

#### 执行项

- 正式定义前台页面使用的背景、表面、文字、边框、强调色语义 token
- 把前台常用 `bg-*`、`text-*`、`border-*` 的散写方案逐步替换为语义化变量
- 补齐前台页面和内容页的 surface 层级规则
- 明确亮色模式优先、暗黑模式克制的落地标准

#### 交付标准

- 首页、文章页、分类页、专题页的表面层级风格统一
- 前台明暗模式切换后保持品牌一致性
- 前台样式维护从“页面级调色”转向“系统级调色”

## 四、本轮建议先做的任务包

基于当前阶段，建议下一轮开发直接按以下顺序推进：

1. 先补齐前台共用视觉规范文档
2. 再统一前台字体系统与标题层级
3. 接着重构前台内容卡片与列表版式
4. 然后对文章详情页做一轮正文节奏精修
5. 最后收口 token 与全站细节

## 五、与文档体系的联动要求

本阶段涉及长期复用的规则，不能只停留在页面实现里，必须同步到文档体系中。

需要同步文档化的内容包括：

- 前台品牌字体与标题层级
- 前台页面留白与栅格规则
- 前台图片使用与封面裁切规则
- 前台内容卡片与内容块的版式层级
- 前台页面动效和 hover 反馈边界
- 前台语义化视觉 token

对应文档建议如下：

- 本文档负责记录“做什么”和“按什么顺序做”
- `ui/` 子目录中的规范文档负责记录“具体怎么统一”

## 六、当前正式结论

当前前台视觉升级阶段正式约定如下：

- 下一阶段优先做“前台品牌系统升级”和“共用视觉规范落地”
- 前台优化重点不再是继续堆页面，而是统一视觉语言、提升阅读沉浸感
- 所有会跨页面复用的规则，必须先补到文档后再沉淀实现
- 后续首页、分类页、专题页、文章页的改版，都应以本文档和共用规范为共同依据
