# 封面古诗词打字机组件接入方案

本文档用于说明 `Yunyu` 前台封面区古诗词打字机能力的组件设计、接口来源、接入页面与后续维护约定，作为文章详情页、分类页、专题页统一实施依据。

## 一、目标

在文章详情页、分类页、专题页的封面中央增加一条由客户端实时请求的古诗词名句，并通过打字机动效呈现，提升首屏停留感、阅读气质与内容站品牌辨识度。

本能力需要满足以下目标：

- 前端公共组件实现，避免每个页面重复接入接口
- 仅在客户端请求，不走服务端转发
- 页面只负责挂载组件，不负责管理接口细节
- 请求失败时提供本地兜底文案，避免封面出现空白
- 与现有 `YunyuHero`、`PostCoverHero` 保持统一封面结构

## 二、接口选型

当前接入方案选择 `今日诗词` 官方浏览器 SDK。

选择原因：

- 官方明确提供网站快速安装与浏览器端高级调用方式
- 支持在浏览器环境中获取诗句、作者、朝代、作品标题
- 官方明确说明浏览器场景应优先使用 SDK，而不是自行从服务端中转
- 适合本项目“封面一行诗句”这种轻量、强调氛围的使用场景

官方接入信息：

- SDK 脚本地址：`https://sdk.jinrishici.com/v2/browser/jinrishici.js`
- 浏览器推荐接入方式：调用 `jinrishici.load(callback)`
- 当前能力只在浏览器端调用，不参与 SSR 首屏数据拼装

## 三、组件设计

公共组件名称：

- `YunyuPoetryTypewriter`

组件目录：

- `yunyu-web/app/components/common/YunyuPoetryTypewriter.vue`

组件职责：

- 在客户端加载 `今日诗词` 浏览器 SDK
- 请求一条诗句与出处信息
- 执行打字机逐字显示动画
- 在请求失败时回退到本地内置诗句
- 根据页面主色切换不同视觉变体

支持参数：

- `variant`
  - `sky`：专题页封面使用
  - `orange`：分类页封面使用
  - `neutral`：文章详情页封面使用
- `align`
  - 控制组件内容居中或左对齐

## 四、Hero 接入方式

为了不破坏现有封面组件的底部信息区结构，`YunyuHero` 与 `PostCoverHero` 增加 `center` 插槽。

插槽职责：

- 专门承接封面中央的轻量氛围内容
- 不替代原有底部标题、标签、统计信息
- 页面层通过 `#center` 注入古诗词组件

这样可以让封面结构分为两层：

- 中央层：古诗词打字机
- 底部层：页面标题、分类标签、文章元信息

## 五、接入页面

当前首批接入页面：

- 文章详情页：`yunyu-web/app/pages/posts/[slug].vue`
- 分类列表页：`yunyu-web/app/pages/categories/index.vue`
- 专题列表页：`yunyu-web/app/pages/topics/index.vue`

接入规则：

- 古诗词组件固定放在封面中央区域
- 页面原有封面主信息继续保留在底部
- 诗句属于氛围增强信息，不承担主标题职责

## 六、客户端请求约束

本功能必须遵循以下约束：

- 不在 Nuxt 服务端渲染阶段请求诗词接口
- 不通过后端接口代理转发诗词请求
- 不在页面顶层 `useAsyncData` 中调用该接口
- 组件必须在 `mounted` 后再请求

原因：

- 官方文档明确说明网站场景应使用浏览器 SDK
- 诗词推荐依赖客户端环境信息与 Token/Cookie 机制
- 如果改为服务端调用，会影响推荐质量与接口使用方式

## 七、失败兜底策略

为避免外部接口波动影响封面展示，组件内置本地兜底诗句列表。

兜底触发条件：

- SDK 脚本加载失败
- SDK 未正确挂载到浏览器全局对象
- `load` 请求失败

兜底要求：

- 至少保留 3 条适合封面气质的古诗词
- 兜底数据结构与接口返回结构保持一致
- 即使接口不可用，打字机效果也继续执行

## 八、视觉与交互规则

封面诗句的视觉规则：

- 使用半透明玻璃感容器承接文本
- 宽度不超过主视觉安全区，避免覆盖底部主标题
- 文本在移动端与桌面端都要可读
- 保持轻量高光，不压过页面主内容

打字机规则：

- 默认按字符逐字输出
- 标点符号处适当增加停顿
- 保留闪烁光标，增强动态反馈
- 动画节奏以“安静、克制”为主，不做夸张跳动

## 九、后续扩展建议

后续可继续扩展：

- 支持“换一句”交互
- 支持在标签页、分类详情页、专题详情页复用
- 支持后台配置是否启用封面古诗词
- 支持不同页面设定不同默认兜底诗句池

## 十、落地结果

本次落地完成后，前台首屏封面将形成统一能力：

- `YunyuHero` / `PostCoverHero` 负责封面骨架
- `YunyuPoetryTypewriter` 负责古诗词氛围内容
- 页面层只负责决定是否接入以及使用哪种视觉变体
