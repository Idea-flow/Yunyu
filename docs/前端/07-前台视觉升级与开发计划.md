# 前台视觉升级与开发计划

## 文档目标

本文档用于明确 `Yunyu` 前台站点从“已可用内容站”升级为“个人最美博客 / 最美个人内容站”的具体建设路径、优先级与阶段性交付物。

本文档不是泛泛的审美讨论，而是后续实际开发的执行依据。

## 一、总体目标

`Yunyu` 前台后续的核心目标，不是继续堆积页面，而是建立一个具备以下特征的内容站：

- 首页具有强品牌记忆点
- 文章详情页具备沉浸式阅读体验
- 分类、专题、标签不是“列表页”，而是内容入口页
- 图片、标题、摘要、空状态、加载态形成统一的视觉语言
- 明亮模式优雅、暗黑模式克制、移动端依然完整

最终目标是让用户进入站点后的第一感受不是“一个博客模板”，而是“一个有明确气质的个人内容品牌”。

## 二、当前现状判断

结合当前首页与现有前台页面，当前项目已经具备以下基础：

- 已完成前后台分层
- 已建立主题系统
- 已有前台首页、分类、专题、文章详情等核心页面
- 已开始建立公共组件体系，例如 `YunyuImage`

但当前距离“最美个人内容站”仍有以下明显差距：

- 首页结构仍偏信息聚合，品牌首屏不够强
- 页面卡片感仍然较重，内容站气质还不够纯净
- 文章详情页阅读沉浸感还不够强
- 分类页、专题页更像普通入口列表，缺少主题氛围
- 公共视觉组件体系仍处于起步阶段
- 视觉层级、动效语言、图片叙事还没有完全统一

## 三、视觉总论

### 3.1 Visual Thesis

`Yunyu` 的前台视觉应该呈现为：

- 像一本被认真策划的数字杂志
- 以大图、留白、标题层级和节制动效建立气质
- 不是卡片拼装站，而是内容与情绪共同构成的阅读场

### 3.2 Content Plan

前台首页后续统一采用以下结构思路：

1. Hero：品牌、主张、主打内容、主视觉
2. Support：推荐内容与当前重点阅读路径
3. Detail：分类与专题入口，强调“如何逛”
4. Final Flow：最新内容流与继续阅读入口

### 3.3 Interaction Thesis

前台页面动效后续统一优先做以下三类：

- 首屏层级渐显与图片轻微推进
- 图片 hover 的轻量缩放与明暗过渡
- 分区进入时的弱位移和透明度过渡

动效目标是增强气质与层级，不做炫技型动效。

## 四、开发优先级

### P0：首页视觉升级

这是当前最优先的部分，也是最直接决定“第一眼是否高级”的页面。

P0 目标：

- 把首页首屏改成更强的品牌化内容首页
- 降低首页中段的卡片堆积感
- 强化主打文章、推荐阅读、分类入口、专题入口之间的节奏
- 建立更明确的版式主次关系

P0 需要做的内容：

- 重构首页 hero 结构
- 加强品牌标题、主文案、摘要和行动入口
- 优化主打文章图文比例与叠层
- 优化推荐内容区的层级与留白
- 优化分类 / 专题入口的视觉表达方式

P0 当前已完成：

- 首页首屏已重构为品牌文案 + 主打内容大图的杂志式 Hero
- 首页首屏已进一步调整为“导航栏下方直接展示大图背景，并叠加轻量星空氛围层”的大图 Hero 结构
- 首页已引入更明确的阅读路径结构，包括推荐阅读、分类地图、专题路线、最新内容
- 首页中段已明显降低“卡片墙”感，转向更强的分区节奏与内容导览感
- 首页分区标题已抽象为公共组件 `YunyuSectionTitle`

### P1：文章详情页沉浸式升级

文章详情页决定站点是否真的适合“长期阅读”。

P1 目标：

- 提升文章首屏的仪式感
- 提升正文排版的节奏感
- 提升目录、引用、图片、代码块、相关文章的体验

P1 需要做的内容：

- 重做文章头图区与标题区
- 优化正文宽度、字重、字号、段落节奏
- 优化目录吸附与阅读辅助信息
- 优化相关文章与延伸阅读模块

P1 当前已完成：

- 文章详情页头部已重构为更具封面感的图文首屏
- 文章详情页首部已进一步调整为“导航栏下方直接展示大封面背景，标题与元信息压在图片底部”的大图 Hero 结构
- 作者信息、发布时间、阅读时长、浏览量与喜欢数已统一为阅读元信息区
- 正文阅读容器已扩大留白并强化阅读区层级
- 相关文章区已改为更完整的延伸阅读区
- 目录侧栏已支持随正文滚动自动高亮
- 顶部已增加阅读进度条
- 代码高亮主题已改为随明暗模式自动切换

P1 下一步待做：

- 继续精修正文排版细节，包括段落节奏、图片插入后的留白与标题间距
- 继续优化目录吸附与移动端目录体验
- 继续强化相关文章与标签、专题之间的跳转闭环

### P2：分类 / 专题 / 标签入口页升级

这些页面不应只是“筛选结果”，而应成为不同阅读路径的入口。

P2 目标：

- 让分类页、专题页、标签页更具主题气质
- 降低纯列表感
- 强化入口页与内容站整体风格的一致性

P2 需要做的内容：

- 提升头图区表达
- 优化入口卡片版式
- 增加更清晰的导览信息
- 统一入口页的信息密度与视觉节奏

### P3：公共视觉组件体系补齐

没有公共视觉组件体系，后续页面会越来越散。

P3 目标：

- 建立基础公共视觉件
- 统一图片、头像、空状态、加载态、标题区、内容分组的表现

P3 建议组件：

- `YunyuAvatar`
- `YunyuEmpty`
- `YunyuLoading`
- `YunyuSectionTitle`
- `YunyuHero`
- `YunyuContentBlock`

### P4：细节体验与性能收尾

当核心视觉与阅读体验完成后，再统一处理收尾质量。

P4 目标：

- 强化移动端体验
- 完善暗黑模式细节
- 优化图片加载与首屏观感
- 优化页面切换和细小交互

## 五、阶段计划

### 第一阶段：首页升级

阶段目标：

- 先完成首页从“可用”到“惊艳”的转变

交付物：

- 新版首页首屏
- 新版推荐阅读区
- 新版分类 / 专题入口区
- 首页样式节奏统一

当前进度：

- 已完成第一阶段主要结构升级
- 当前首页已经可以作为后续前台页面的视觉参考样板

### 第二阶段：文章详情页升级

阶段目标：

- 建立真正适合长文阅读的详情页体验

交付物：

- 新版文章头部
- 新版正文视觉规则
- 目录与延伸阅读升级

当前进度：

- 已完成文章详情页第一轮与第二轮升级
- 当前已具备首屏重构、正文容器升级、目录联动、阅读进度、相关文章升级等基础能力
- 下一轮将进入正文排版精修与目录体验细化

### 第三阶段：入口页统一升级

阶段目标：

- 让分类、专题、标签形成统一入口语言

交付物：

- 分类列表页升级
- 专题列表页升级
- 标签页升级
- 分类 / 专题 / 标签详情页升级

### 第四阶段：公共组件与细节收尾

阶段目标：

- 让前台所有页面回到统一组件与统一节奏之下

交付物：

- 公共视觉组件补齐
- 空态、加载态统一
- 暗黑模式与移动端细节收尾

## 六、当前开发决策

本轮开发从以下内容开始：

1. 先完成本文档
2. 立即开始首页第一阶段视觉升级
3. 当前以首页为 P0，不先分散修改全部页面

原因如下：

- 首页是品牌第一印象
- 首页改好后可以反向定义后续页面视觉标准
- 首页中的视觉语言更适合先沉淀为公共模式

## 七、实施约束

后续开发必须遵循以下约束：

- 前台避免回到模板化卡片堆积
- 优先通过版式、图片、留白、标题层级建立高级感
- 公共能力优先沉淀到 `common/`
- 公共组件引用统一使用 `~/components/common/...`
- 用户界面不出现研发说明文字
- 明亮模式优先表达品牌，暗黑模式负责长时间舒适阅读

## 八、当前结论

`Yunyu` 后续要成为“个人最美博客 / 最美个人内容站”，当前最应该优先做的不是补更多页面，而是：

- 先把首页做到足够惊艳
- 再把文章详情页做到足够沉浸
- 再统一分类、专题、标签的入口体验
- 最后补齐公共视觉组件与细节质量

当前正式执行顺序为：

1. 首页视觉升级
2. 文章详情页升级
3. 分类 / 专题 / 标签入口升级
4. 公共视觉组件补齐与细节收尾

## 九、当前已落地改动记录

为了避免后续开发时忘记“之前为什么这么改”，当前已明确记录以下事实：

- 首页已经完成第一阶段视觉升级，不再回退为普通信息聚合页
- 首页首屏当前正式采用“大图背景 + 星空氛围层”的 Hero 方向
- 文章详情页已经开始从“展示页”转向“沉浸式阅读页”
- `YunyuImage` 已承担前台图片统一能力
- `YunyuSectionTitle` 已承担首页和文章详情页的分区标题统一能力
- `YunyuHero` 已承担首页和文章详情页的大图首屏统一能力
- 公共组件导入统一使用 `~/components/common/...`
- 文章详情页首部当前正式采用“大图背景 Hero”方向，后续不再回到普通左右分栏头图
- 首页与文章详情页当前正式采用“导航栏覆盖首屏”的模式，后续不再回到首屏上方硬切分导航

当前这些改动已经不再是临时尝试，而是前台长期视觉方向的一部分。

## 十、下一轮建议

下次继续开发时，建议直接从以下顺序接续：

1. 继续细化文章详情页正文排版与目录体验
2. 开始统一分类 / 专题 / 标签入口页视觉语言
3. 继续补齐 `YunyuAvatar`、`YunyuEmpty`、`YunyuLoading` 等基础组件
