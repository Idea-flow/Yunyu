# 云屿第一版部署执行步骤

## 文档目标

本文档用于把 `云屿 / Yunyu` 第一版部署方案拆成实际执行步骤，方便按顺序落地。

## 一、仓库内已落地内容

当前仓库已补充以下部署基础文件：

- 根目录 `docker-compose.yml`
- 根目录 `.env.deploy.example`
- GitHub 工作流 `.github/workflows/build-yunyu-server-image.yml`
- 后端镜像文件 `yunyu-server/Dockerfile`
- 后端生产配置 `yunyu-server/src/main/resources/application-prod.yml`

## 二、执行步骤

### 步骤 1：准备 GitHub 镜像仓库

1. 将代码仓库推送到 GitHub
2. 确认仓库开启 `GitHub Actions`
3. 确认组织或个人账户允许使用 `GHCR`
4. 首次推送后检查工作流是否成功生成镜像

镜像默认命名规划：

- `ghcr.io/<github-owner>/yunyu-server:latest`
- `ghcr.io/<github-owner>/yunyu-server:sha-xxxxxxx`

### 步骤 2：调整服务器部署变量

1. 在服务器部署目录复制环境文件：

```bash
cp .env.deploy.example .env
```

2. 至少修改以下变量：

```bash
YUNYU_SERVER_IMAGE=ghcr.io/<github-owner>/yunyu-server:latest
MYSQL_PASSWORD=请替换成强密码
MYSQL_ROOT_PASSWORD=请替换成强密码
YUNYU_JWT_SECRET=请替换成正式密钥
```

### 步骤 3：登录镜像仓库并启动后端

如果镜像仓库是私有的，需要先在服务器登录 `GHCR`：

```bash
docker login ghcr.io
```

然后启动服务：

```bash
docker compose pull
docker compose up -d
```

查看状态：

```bash
docker compose ps
docker compose logs -f yunyu-server
```

### 步骤 4：验证后端服务

检查后端健康接口：

```bash
curl http://127.0.0.1:20000/actuator/health
```

检查接口文档：

```bash
http://服务器IP:20000/swagger-ui.html
```

### 步骤 5：部署前端到 Cloudflare Pages

1. 在 Cloudflare Pages 中连接 GitHub 仓库
2. 将项目根目录设置为仓库根目录
3. 将构建目录设置为 `yunyu-web`
4. 安装命令填写：

```bash
pnpm install
```

5. 构建命令填写：

```bash
pnpm build
```

6. 输出目录按 Cloudflare Pages 当前 Nuxt 适配方式配置

说明：

- 如果采用 `@cloudflare/nuxt` 方案，可继续补充 Pages 适配配置
- 如果当前只需要先上线静态页面，可评估 `pnpm generate`
- 由于当前仓库还未补 Cloudflare 专属适配文件，前端上线步骤先按“接入与配置”作为下一步实施项

### 步骤 6：配置前端环境变量

在 Cloudflare Pages 项目中配置：

```bash
NUXT_PUBLIC_API_BASE=https://api.你的域名
```

如果前后端还未挂域名，临时可以先写：

```bash
NUXT_PUBLIC_API_BASE=http://你的服务器IP:20000
```

### 步骤 7：联调前后端

1. 打开前端页面
2. 检查前端请求是否成功访问后端
3. 检查登录接口是否正常返回 Token
4. 检查内容列表、详情页、后台接口是否联通

## 三、建议的分步执行节奏

### 第一轮

- 先让 GitHub Actions 成功产出后端镜像
- 先让服务器上的 `Spring Boot + MySQL` 跑起来
- 先确认接口联通与数据库初始化成功

### 第二轮

- 接入 Cloudflare Pages
- 填写前端环境变量
- 验证页面访问和 API 调用

### 第三轮

- 绑定正式域名
- 增加反向代理
- 增加 HTTPS、备份与监控

## 四、当前注意事项

- `docker-compose.yml` 中的默认镜像地址是示例值，上线前必须替换
- 默认数据库密码和 JWT 密钥仅用于示例，正式环境必须修改
- 当前前端还未补充 Cloudflare Pages 专属构建适配文件，这一步建议作为下一阶段继续实施
- 当前后端默认直接暴露 `20000` 端口，正式环境建议通过反向代理统一收口
