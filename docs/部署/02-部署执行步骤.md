# 云屿第一版部署执行步骤

## 文档目标

本文档用于把 `云屿 / Yunyu` 第一版部署方案拆成实际执行步骤，方便按顺序落地。

## 一、仓库内已落地内容

当前仓库已补充以下部署基础文件：

- 根目录 `docker-compose.yml`
- 根目录 `.env.deploy.example`
- GitHub 工作流 `.github/workflows/build-yunyu-server-image.yml`
- 后端镜像文件 `yunyu-server/Dockerfile`
- 后端生产配置 `yunyu-server/src/main/resources/application-prod.yml`

## 二、执行步骤

### 步骤 1：准备 GitHub 镜像仓库

1. 将代码仓库推送到 GitHub
2. 确认仓库开启 `GitHub Actions`
3. 确认组织或个人账户允许使用 `GHCR`
4. 首次推送后检查工作流是否成功生成镜像

镜像默认命名规划：

- `ghcr.io/<github-owner>/yunyu-server:latest`
- `ghcr.io/<github-owner>/yunyu-server:sha-xxxxxxx`

### 步骤 2：调整服务器部署变量

1. 在服务器部署目录复制环境文件：

```bash
cp .env.deploy.example .env
```

2. 至少修改以下变量：

```bash
YUNYU_SERVER_IMAGE=ghcr.io/<github-owner>/yunyu-server:latest
MYSQL_PASSWORD=请替换成强密码
MYSQL_ROOT_PASSWORD=请替换成强密码
YUNYU_JWT_SECRET=请替换成正式密钥
```

### 步骤 3：登录镜像仓库并启动后端

如果镜像仓库是私有的，需要先在服务器登录 `GHCR`：

```bash
docker login ghcr.io
```

启动前请确认项目根目录下的 MySQL 数据目录存在或可被 Docker 自动创建：

```bash
./yunyu_mysql_data
```

当前 `docker-compose.yml` 已将该目录绑定到 MySQL 容器内的 `/var/lib/mysql`，用于持久化数据库文件。

然后启动服务：

```bash
docker compose pull
docker compose up -d
```

查看状态：

```bash
docker compose ps
docker compose logs -f yunyu-server
```

### 步骤 4：验证后端服务

检查后端健康接口：

```bash
curl http://127.0.0.1:20000/actuator/health
```

检查接口文档：

```bash
http://服务器IP:20000/swagger-ui.html
```

### 步骤 5：在 1Panel 中反向代理后端

第一版后端先不单独补 `Nginx` 配置文件，而是直接使用 `1Panel` 反向代理：

1. 确保服务器 `20000` 端口上的 `yunyu-server` 已正常启动
2. 在 `1Panel` 中新增网站或反向代理配置
3. 将域名例如 `api.你的域名` 指向服务器
4. 反向代理目标填写：

```bash
http://127.0.0.1:20000
```

5. 如需跨域，前端域名与后端域名确定后再做联调验证
6. HTTPS 证书先直接在 `1Panel` 中申请和绑定

建议第一版域名拆分：

- 前端：`www.你的域名`
- 后端：`api.你的域名`

### 步骤 6：部署前端到 Cloudflare Pages

这一步如果你使用 Cloudflare 后台页面手动创建项目，当前可以直接走 Git 集成，不一定要先补 Cloudflare 专属适配代码。

根据 Nuxt 官方文档，Cloudflare Pages 的 Git 集成场景下通常可以零配置识别 Nuxt；根据 Cloudflare 官方 Nuxt 页面，常见构建配置可直接使用 `npm run build` 和 `dist` 目录。结合你当前仓库使用 `pnpm`，第一版建议直接在 Cloudflare 页面中手动填写 `pnpm` 的安装和构建命令。

在 Cloudflare 后台手动操作流程如下：

1. 登录 Cloudflare 控制台
2. 进入 `Workers & Pages`
3. 点击 `Create application`
4. 选择 `Pages`
5. 选择 `Import an existing Git repository`
6. 授权并选择你的 GitHub 仓库
7. 创建项目时填写以下构建信息

推荐填写值：

- Project name：`yunyu-web` 或你的正式站点名
- Production branch：`main`
- Root directory：`yunyu-web`
- Build command：`pnpm build`
- Build output directory：`dist`

如果页面里有安装命令或包管理器相关选项，填写：

```bash
pnpm install
```

构建命令填写：

```bash
pnpm build
```

输出目录填写：

```bash
dist
```

说明：

- 当前这版优先走 Cloudflare Pages 的 Git 集成部署
- 如果后续发现 Pages 构建环境无法正确识别当前 Nuxt 输出，再补充 Cloudflare 专属适配配置
- 如果后续改成纯静态部署，也可以改为 `pnpm generate`

### 步骤 7：配置前端环境变量

在 Cloudflare Pages 项目中配置：

```bash
YUNYU_PUBLIC_API_BASE=https://api.你的域名
```

如果前后端还未挂域名，临时可以先写：

```bash
YUNYU_PUBLIC_API_BASE=http://你的服务器IP:20000
```

建议第一版直接填写反代后的后端地址：

```bash
YUNYU_PUBLIC_API_BASE=https://api.你的域名
```

### 步骤 8：联调前后端

1. 打开前端页面
2. 检查前端请求是否成功访问后端
3. 检查登录接口是否正常返回 Token
4. 检查内容列表、详情页、后台接口是否联通

## 三、建议的分步执行节奏

### 第一轮

- 先让 GitHub Actions 成功产出后端镜像
- 先让服务器上的 `Spring Boot + MySQL` 跑起来
- 先确认接口联通与数据库初始化成功

### 第二轮

- 接入 Cloudflare Pages
- 填写前端环境变量
- 验证页面访问和 API 调用

### 第三轮

- 绑定正式域名
- 将反向代理方案从 `1Panel` 固化为独立 `Nginx` 配置
- 增加 HTTPS、备份与监控

## 四、当前注意事项

- `docker-compose.yml` 中的默认镜像地址是示例值，上线前必须替换
- 默认数据库密码和 JWT 密钥仅用于示例，正式环境必须修改
- 第一版后端反代由 `1Panel` 承担，独立 `Nginx` 部署方案放到第二版
- 当前前端可以先直接走 Cloudflare Pages 后台手动接 GitHub 的方式部署，如构建失败再针对 Pages 补适配配置
- MySQL 数据当前持久化在项目目录下的 `./yunyu_mysql_data`，不要误删该目录
