# 云屿 / Yunyu 文档中心

本目录用于沉淀 `云屿 / Yunyu` 从个人内容站到内容平台阶段的全部文档，确保产品、技术、部署与实施记录长期可维护。

## 目录说明

- `architecture/`：系统架构、技术选型、部署方案、认证方案、数据流设计。
- `adr/`：Architecture Decision Record，重要技术决策记录。
- `product/`：产品规划、信息架构、页面说明、业务流程、商品与虚拟产品方案。
- `tech/`：接口规范、数据库设计、模块设计、开发规范、测试方案。
- `standards/`：文档治理、命名规范、变更同步要求。
- `operations/`：部署、运维、监控、备份、告警、发布流程。

## 文档编写原则

- 所有核心决策优先写入文档，再进入实现阶段。
- 任何会影响产品逻辑、接口契约、部署方式、数据模型的变更，都必须同步更新相关文档。
- 新文档优先放到最贴近业务语义的目录中，避免所有内容堆在同一级目录。
- 重要架构决策必须新增一篇 `adr` 记录，不直接埋在聊天记录里。

## 当前主内容技术结论

当前项目已确认：

- 项目中文名：`云屿`
- 项目英文名：`Yunyu`
- 开源仓库与后续代码命名默认统一使用 `Yunyu`

- 平台主内容存放于服务端，不以仓库内 Markdown 文件作为主数据源
- 内容编辑格式使用 `Markdown`
- 内容发布时由后端转换 HTML 与派生字段
- `Nuxt 4` 负责 SSR 页面输出，确保 SEO
- `@nuxt/content` 仅作为静态文档类能力的可选补充
