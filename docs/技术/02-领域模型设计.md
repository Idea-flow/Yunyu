# 领域模型设计

## 文档目标

本文档用于定义 `云屿 / Yunyu` 从个人内容站演进到内容平台阶段的核心领域对象与职责边界。

## 一、领域划分

本项目建议划分为以下核心领域：

- 用户域
- 内容域
- 商品域
- 交易域
- 媒体域
- 运营域

## 二、内容域

内容域是当前阶段核心。

### 2.1 核心实体

- `Post`
- `PostContent`
- `PostContentVersion`
- `Category`
- `Tag`
- `Topic`
- `Author`

### 2.2 实体职责

#### `Post`

负责内容的基础元信息与业务状态：

- 标题
- 摘要
- 封面
- slug
- 发布状态
- 作者
- 分类
- SEO 信息
- 发布时间

#### `PostContent`

负责当前生效正文内容与派生结果：

- Markdown 原文
- HTML 内容
- 纯文本
- 目录
- 富内容块
- 阅读时长

#### `PostContentVersion`

负责内容版本能力：

- 历史版本保存
- 发布前版本对比
- 回滚支持

#### `Topic`

负责跨文章的专题聚合。

#### `Author`

负责作者信息展示与内容归属，不强绑定后台账号模型。

## 三、商品域

商品域用于支撑平台从内容站向内容商业化演进。

### 3.1 核心实体

- `Product`
- `ProductSku`
- `DigitalAsset`
- `ProductContent`

### 3.2 设计原则

- 商品与文章分离建模
- 商品可绑定内容
- 虚拟产品可绑定下载资源
- 后续支持商品与专题、文章互相挂接

## 四、交易域

当前阶段只预留，不必一次做重。

### 4.1 核心实体

- `Order`
- `OrderItem`
- `Payment`
- `UserEntitlement`

### 4.2 价值

该领域用于未来支撑：

- 虚拟产品购买
- 用户资产归属
- 下载权限
- 订单追踪

## 五、媒体域

媒体域负责统一管理内容所需的资源。

### 5.1 核心实体

- `MediaFile`
- `MediaGroup`

### 5.2 价值

用于统一管理：

- 封面图
- 正文图片
- 九宫格图片
- 下载附件
- 商品配图

## 六、运营域

运营域负责首页、推荐位、专题推荐、导航配置等内容编排。

### 6.1 核心实体

- `SiteSection`
- `RecommendSlot`
- `Banner`
- `NavigationItem`

## 七、内容详情页查询原则

### 7.1 列表查询

列表页、推荐位、后台列表页应优先查询轻量元数据，不应携带正文大字段。

### 7.2 详情查询

详情页按需查询：

- `Post`
- `PostContent`

这样可以减少频繁查询时的 IO 压力。

## 八、核心表拆分建议

### 8.1 主表：`post`

建议包含：

- `id`
- `title`
- `slug`
- `summary`
- `cover_url`
- `author_id`
- `category_id`
- `status`
- `seo_title`
- `seo_description`
- `published_at`
- `view_count`
- `created_at`
- `updated_at`

### 8.2 内容表：`post_content`

建议包含：

- `id`
- `post_id`
- `content_markdown`
- `content_html`
- `content_plain_text`
- `content_toc_json`
- `content_blocks_json`
- `reading_time`
- `content_version`
- `created_at`
- `updated_at`

### 8.3 版本表：`post_content_version`

建议包含：

- `id`
- `post_id`
- `version_no`
- `content_markdown`
- `content_html`
- `content_blocks_json`
- `operator_id`
- `created_at`

## 九、当前阶段正式建议

当前阶段推荐优先实现以下领域对象：

- `Post`
- `PostContent`
- `Category`
- `Tag`
- `Topic`
- `Author`
- `MediaFile`

第二阶段再逐步扩展：

- `Product`
- `DigitalAsset`
- `Order`
- `UserEntitlement`
