# 后端结构与依赖说明

## 文档目标

本文档用于说明 `yunyu-server` 当前阶段的项目目录结构、各目录职责以及 `pom.xml` 中核心依赖的用途，作为后端开发启动基线文档。

## 一、项目基础信息

当前后端项目基线如下：

- 项目目录：`yunyu-server`
- 技术栈：`Java 25 + Spring Boot 4`
- 项目形态：单体项目
- 构建方式：`Maven`
- 包名：`com.ideaflow.yunyu`
- 接口文档路线：`OpenAPI3`

## 二、为什么当前使用单体项目

当前阶段不建议直接拆 Maven 多模块，原因如下：

- 当前业务仍处于第一阶段，核心是快速建立内容平台后端基础能力
- 文章、分类、标签、专题、评论、站点配置等模块仍高度相关
- 单体项目更容易启动、调试、联调和部署
- 先在代码层做模块化分包，后续再决定是否拆成 Maven 多模块

当前阶段结论：

- 使用一个 `pom.xml`
- 使用一个 Spring Boot 启动入口
- 在代码层按业务模块分包

### 3.2 `yunyu-server/` 目录说明

当前已存在的核心文件与目录如下：

```text
yunyu-server/
├── .mvn/
├── mvnw
├── mvnw.cmd
├── pom.xml
└── src/
```

各目录与文件作用：

- `.mvn/`
  Maven Wrapper 配置目录，用于保证不同机器可以使用统一方式执行 Maven 命令
- `mvnw`
  Linux / macOS 下的 Maven Wrapper 启动脚本
- `mvnw.cmd`
  Windows 下的 Maven Wrapper 启动脚本
- `pom.xml`
  Maven 项目核心构建文件，负责依赖管理、插件配置和构建参数
- `src/`
  Java 源码、资源文件与测试代码目录

### 3.3 `src/` 目录说明

推荐理解为：

```text
src/
├── main/
│   ├── java/
│   └── resources/
└── test/
    └── java/
```

目录作用说明：

- `src/main/java/`
  存放正式业务代码
- `src/main/resources/`
  存放配置文件、Mapper XML、静态资源与模板资源
- `src/test/java/`

当前基础配置相关资源文件已落地：

- `src/main/resources/application.yml`
  存放全局应用基础配置与默认激活环境
- `src/main/resources/application-dev.yml`
  存放开发环境 `spring.datasource` 数据库连接、OpenAPI3 与 Actuator 配置骨架

## 四、推荐的后端代码包结构

当前阶段建议采用“单体项目 + 模块化分包”结构：

```text
com.ideaflow.yunyu
├── YunyuServerApplication
├── common
├── config
├── infrastructure
├── security
└── module
    ├── auth
    ├── user
    ├── post
    ├── category
    ├── tag
    ├── topic
    ├── comment
    ├── site
    └── system
```

各目录作用：

- `YunyuServerApplication`
  Spring Boot 启动类，负责启动整个后端应用
- `common`
  存放通用常量、统一返回、异常定义、工具类等公共能力
- `config`
  存放全局配置类，例如 Web 配置、Jackson 配置、跨域配置等
- `infrastructure`
  存放基础设施相关代码，例如 OpenAPI 配置、持久层通用配置
- `security`
  存放认证与权限相关代码，例如安全配置、登录上下文、鉴权逻辑
- `module`
  存放具体业务模块

其中 `module/system/init` 建议专门用于放置：

- 启动自动初始化
- 数据库引导检查
- 建库建表
- 初始化状态判断

### 4.2 业务模块内部建议结构

以 `post` 模块为例，建议结构如下：

```text
module/post/
├── controller
├── service
├── mapper
├── entity
├── dto
├── vo
└── convert
```

各目录作用：

- `controller`
  对外暴露 HTTP 接口
- `service`
  编排业务逻辑
- `mapper`
  定义 MyBatis / MyBatis-Plus 数据访问接口
- `entity`
  对应数据库表实体
- `dto`
  接收请求参数或服务层传输对象
- `vo`
  返回给前端的视图对象
- `convert`
  负责实体、DTO、VO 之间的转换

## 七、当前阶段结论

`yunyu-server` 当前应采用以下策略：

- 一个单体 Spring Boot 项目
- 一个 `pom.xml`
- 代码层模块化分包
- 使用 OpenAPI3 作为接口文档标准
- 使用 `spring.datasource` 原生配置写数据库连接信息
- 应用启动时自动检查数据库、自动建库、自动建表
- 使用 `system` 模块承担启动自动 bootstrap 能力
- 初始化时自动补默认管理员账号：`yunyu / yunyu`
- 初始化时自动补默认管理员的 `LOCAL` 登录方式绑定
- 已初始化时自动跳过，不重复初始化
- 启动日志中明确输出数据库存在性、建库动作、建表动作与跳过信息
- 认证方案第一版使用 `JWT Bearer Token`
- 认证通过后将当前用户写入 `SecurityContextHolder`
- 当前提供 `POST /api/auth/login` 与 `GET /api/auth/me` 两个基础认证接口

该结构适合当前项目快速启动，并保留后续演进空间。
