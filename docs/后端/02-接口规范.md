# 接口规范

## 文档目标

本文档用于约定 `Yunyu` 后端当前阶段已经落地的核心接口风格、统一返回结构和后台管理接口边界，便于前后端联调与后续模块扩展保持一致。

## 一、统一响应结构

当前所有接口统一返回：

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {}
}
```

约定如下：

- `code = 0` 表示业务成功
- 非 `0` 表示业务失败
- `message` 用于返回统一提示语
- `data` 为业务数据载荷
- 对于可以明确识别的错误，`message` 必须返回可读原因，例如邮箱格式不正确、密码长度不合法、请求体格式不正确、资源不存在等

## 二、认证与权限接口

当前已落地的认证接口：

- `POST /api/auth/login`
- `GET /api/auth/me`

接口说明：

- 登录接口使用账号密码登录
- 当前用户接口要求携带 `Bearer Token`
- 后台接口统一使用 `Authorization: Bearer <token>` 访问

## 三、后台接口前缀约定

后台管理类接口统一放在：

- `/api/admin/**`

当前安全规则为：

- 访问 `/api/admin/**` 需要 `SUPER_ADMIN`

这样可以保证站长后台接口和前台公开接口在语义与权限层上保持分离。

## 四、后台用户管理接口

当前已落地后台用户管理接口：

- `GET /api/admin/users`
- `GET /api/admin/users/{userId}`
- `POST /api/admin/users`
- `PUT /api/admin/users/{userId}`
- `DELETE /api/admin/users/{userId}`

### 4.1 查询用户列表

`GET /api/admin/users`

支持查询参数：

- `keyword`：按邮箱或用户名模糊搜索
- `role`：按角色筛选，当前支持 `SUPER_ADMIN`、`USER`
- `status`：按状态筛选，当前支持 `ACTIVE`、`DISABLED`
- `pageNo`：当前页码，默认 `1`
- `pageSize`：每页条数，默认 `10`，最大 `50`

响应数据说明：

- `list`：用户列表
- `total`：当前结果总数
- `pageNo`：当前页码
- `pageSize`：每页条数
- `totalPages`：总页数

### 4.2 查询用户详情

`GET /api/admin/users/{userId}`

用于进入编辑流程前获取单个用户当前资料。

### 4.3 创建用户

`POST /api/admin/users`

请求字段：

- `email`
- `userName`
- `avatarUrl`
- `password`
- `role`
- `status`

当前规则：

- 邮箱必须唯一
- 密码长度需在 `6-32` 位之间
- 未传角色时默认 `USER`
- 未传状态时默认 `ACTIVE`
- 创建成功后会同步写入本地登录绑定 `LOCAL`

### 4.4 更新用户

`PUT /api/admin/users/{userId}`

请求字段：

- `email`
- `userName`
- `avatarUrl`
- `password`
- `role`
- `status`

当前规则：

- 邮箱必须唯一
- 若密码为空，则保持原密码不变
- 更新邮箱时会同步更新 `user_auth` 中的本地绑定信息

### 4.5 删除用户

`DELETE /api/admin/users/{userId}`

当前规则：

- 采用逻辑删除
- 不允许删除当前登录账号
- 若目标为最后一个站长账号，则禁止删除

## 五、前后端联调建议

前端当前建议：

- 通过统一接口层访问后端
- 后端基础地址由 `NUXT_PUBLIC_API_BASE` 控制
- 后台管理页默认使用用户登录态中的 `Bearer Token`

这样后续切换开发、测试、线上环境时，只需要调整环境变量，而不需要修改页面代码。

## 六、后台文章管理接口

当前已落地后台文章管理接口：

- `GET /api/admin/posts`
- `GET /api/admin/posts/{postId}`
- `POST /api/admin/posts`
- `PUT /api/admin/posts/{postId}`
- `DELETE /api/admin/posts/{postId}`

### 6.1 查询文章列表

`GET /api/admin/posts`

支持查询参数：

- `keyword`：按标题或 slug 模糊搜索
- `status`：按状态筛选，当前支持 `DRAFT`、`PUBLISHED`、`OFFLINE`
- `pageNo`：当前页码，默认 `1`
- `pageSize`：每页条数，默认 `10`，最大 `50`

响应数据说明：

- `list`：文章列表
- `total`：当前结果总数
- `pageNo`：当前页码
- `pageSize`：每页条数
- `totalPages`：总页数

### 6.2 查询文章详情

`GET /api/admin/posts/{postId}`

用于进入编辑流程前获取单篇文章详情和正文内容。

### 6.3 创建文章

`POST /api/admin/posts`

请求字段：

- `title`
- `slug`
- `summary`
- `coverUrl`
- `status`
- `contentMarkdown`

当前规则：

- `slug` 必须唯一
- 未传状态时默认 `DRAFT`
- 当状态为 `PUBLISHED` 时，自动写入当前发布时间
- 创建时会同步生成 `post_content` 正文记录

### 6.4 更新文章

`PUT /api/admin/posts/{postId}`

请求字段：

- `title`
- `slug`
- `summary`
- `coverUrl`
- `status`
- `contentMarkdown`

当前规则：

- `slug` 必须唯一
- 修改正文时会同步更新 `post_content`
- 当状态切换为 `PUBLISHED` 且此前没有发布时间时，自动写入发布时间
- 当状态切回非发布状态时，清空发布时间

### 6.5 删除文章

`DELETE /api/admin/posts/{postId}`

当前规则：

- 采用逻辑删除
- 删除后文章不会继续出现在后台列表中
