# draw.io 配图实施方案

> 注意：本文件属于历史辅助方案说明。
>
> 当前项目中 draw.io 的正式规则，以项目根目录下的 `AGENTS.md` 为准。
>
> 尤其是正式 `.drawio` 文件的存放位置，现已改为“对应文档所在目录下的 `资源/文档名/drawio/`”，不再以本文件中的旧路径说明为准。

## 一、先回答你的问题

### 1. 我需要什么 skill 才能画图

如果目标是“知识结构图、流程图、对比图、状态流转图”，严格来说不需要额外 skill。

原因很简单：

- 这类图更偏结构化表达。
- `draw.io` 本身已经足够完成。
- 我主要负责的是：梳理图的内容、设计图的结构、确定图插入文档的位置、统一命名和目录规范。

如果你以后希望我生成更偏视觉化、插画化、海报式的位图素材，那时更适合用 `imagegen` 这类能力。

但针对当前这个 Java 面试知识库场景，最合适的方案不是额外 skill，而是：

- `draw.io` 画结构图
- 导出 `png` 或 `svg`
- 插入到对应 Markdown 文档

### 2. 能不能“画图 + 截图 + 插入文档”

可以实现。

但更推荐的工程化方案不是“手工截屏”，而是：

1. 用 `draw.io` 画图
2. 直接导出 `png` 或 `svg`
3. 把导出文件放到统一目录
4. 在 Markdown 中引用图片

因为这种方式相比手工截屏有明显优势：

- 图更清晰
- 可以重复修改
- 命名规范统一
- 后续迁移和维护更方便

所以后续我们默认采用：

- `draw.io 源文件 + 导出图片 + 文档引用`

而不是：

- 手工截图再粘贴

## 三、推荐命名规范

### 1. draw.io 源文件命名

格式建议：

`序号-知识点名称.drawio`

示例：

- `01-进程与线程关系图.drawio`
- `02-用户态与内核态切换图.drawio`
- `03-虚拟内存与分页图.drawio`

### 2. 导出图片命名

格式建议：

`序号-知识点名称.png`

示例：

- `01-进程与线程关系图.png`
- `02-用户态与内核态切换图.png`
- `03-虚拟内存与分页图.png`

## 四、Markdown 插图建议写法

建议使用相对路径引用。

示例：

```md
![进程与线程关系图](../../项目规范/图示资源/操作系统基础/01-进程与线程关系图.png)
```

如果图比较关键，建议在图片上方加一句引导语，例如：

```md
下面这张图可以帮助你理解“进程拥有资源，线程负责执行”：

![进程与线程关系图](../../项目规范/图示资源/操作系统基础/01-进程与线程关系图.png)
```

## 五、哪些知识最适合画图

最适合画图的知识一般有四类：

### 1. 结构关系类

比如：

- 进程和线程的关系
- JVM 内存结构
- Redis 数据结构分类
- Spring 容器结构

### 2. 流程步骤类

比如：

- 用户态切换到内核态流程
- 一次请求的处理流程
- Bean 生命周期
- TCP 三次握手和四次挥手

### 3. 对比类

比如：

- 进程 vs 线程
- BIO vs NIO vs AIO
- 乐观锁 vs 悲观锁
- 聚簇索引 vs 二级索引

### 4. 状态流转类

比如：

- 线程状态流转
- TCP 状态机
- Spring Bean 生命周期
- Redis 主从切换过程
