# 数据库设计

## 文档目标

本文档用于沉淀 `云屿 / Yunyu` 当前阶段最小可行数据库设计，重点覆盖文章内容、内容拆表与后续平台化扩展预留。

## 一、设计原则

- 高频查询字段与大正文分离
- 内容源数据与派生数据并存
- 状态字段明确
- 为后续商品、虚拟产品预留扩展能力

## 二、用户表 `user`

### 2.1 作用

存放前台注册用户与后台站长账号，当前阶段前后台共用一张用户表。

### 2.2 当前确认版 DDL

```sql
CREATE TABLE `user` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '用户ID',
  `email` VARCHAR(128) NOT NULL COMMENT '邮箱，登录账号',
  `user_name` VARCHAR(64) NOT NULL COMMENT '用户名称/展示名称',
  `avatar_url` VARCHAR(255) DEFAULT NULL COMMENT '头像URL',
  `password` VARCHAR(255) NOT NULL COMMENT '密码原值',
  `password_hash` VARCHAR(255) NOT NULL COMMENT '密码哈希值',
  `role` VARCHAR(32) NOT NULL DEFAULT 'USER' COMMENT '角色：SUPER_ADMIN / USER',
  `status` VARCHAR(32) NOT NULL DEFAULT 'ACTIVE' COMMENT '状态：ACTIVE / DISABLED',
  `last_login_at` DATETIME DEFAULT NULL COMMENT '最后登录时间',
  `last_login_ip` VARCHAR(64) DEFAULT NULL COMMENT '最后登录IP',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否删除：0否 1是',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_user_email` (`email`),
  KEY `idx_user_status` (`status`),
  KEY `idx_user_created_at` (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='用户表';
```

### 2.3 当前说明

- `email` 作为登录账号
- `user_name` 作为展示名称
- `role` 当前仅使用 `SUPER_ADMIN` 与 `USER`
- `status` 当前仅使用 `ACTIVE` 与 `DISABLED`
- 当前阶段不单独拆后台用户表

## 三、内容主表命名建议

当前项目建议数据库表使用单数命名，因此内容主表建议命名为：

- `post`

而不是：

- `posts`

原因：

- 和 `user`、`comment`、`category` 这类单数表名更一致
- Java 实体命名更自然，例如 `Post`、`PostContent`
- MyBatis-Plus 映射时语义更统一

如果你后续非常明确想采用全复数风格，也可以统一改成 `users`、`posts`、`comments`，但不要混用。

当前阶段更推荐：

- `user`
- `post`
- `post_content`
- `comment`

## 四、文章主表 `post`

### 4.1 作用

存放文章基础信息与高频查询字段。

### 4.2 当前确认版 DDL

```sql
CREATE TABLE `post` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '文章ID',
  `title` VARCHAR(200) NOT NULL COMMENT '文章标题',
  `slug` VARCHAR(220) NOT NULL COMMENT '文章唯一标识',
  `summary` VARCHAR(500) DEFAULT NULL COMMENT '文章摘要',
  `cover_url` VARCHAR(255) DEFAULT NULL COMMENT '封面图URL',
  `user_id` BIGINT UNSIGNED NOT NULL COMMENT '所属用户ID',
  `category_id` BIGINT UNSIGNED DEFAULT NULL COMMENT '分类ID',
  `status` VARCHAR(32) NOT NULL DEFAULT 'DRAFT' COMMENT '状态：DRAFT / PUBLISHED / OFFLINE',
  `is_top` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否置顶：0否 1是',
  `is_recommend` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否推荐：0否 1是',
  `has_video` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否有视频：0否 1是',
  `allow_comment` TINYINT(1) NOT NULL DEFAULT 1 COMMENT '是否允许评论：0否 1是',
  `seo_title` VARCHAR(255) DEFAULT NULL COMMENT 'SEO标题',
  `seo_description` VARCHAR(500) DEFAULT NULL COMMENT 'SEO描述',
  `published_at` DATETIME DEFAULT NULL COMMENT '发布时间',
  `sort_order` INT NOT NULL DEFAULT 0 COMMENT '排序值',
  `view_count` BIGINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '浏览量',
  `like_count` BIGINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '点赞数',
  `comment_count` BIGINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '评论数',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否删除：0否 1是',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_post_slug` (`slug`),
  KEY `idx_post_user_id` (`user_id`),
  KEY `idx_post_category_id` (`category_id`),
  KEY `idx_post_status` (`status`),
  KEY `idx_post_published_at` (`published_at`),
  KEY `idx_post_sort_order` (`sort_order`),
  KEY `idx_post_created_at` (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='文章主表';
```

### 4.3 推荐字段

- `id`
- `title`
- `slug`
- `summary`
- `cover_url`
- `user_id`
- `category_id`
- `status`
- `is_top`
- `is_recommend`
- `has_video`
- `allow_comment`
- `seo_title`
- `seo_description`
- `published_at`
- `sort_order`
- `view_count`
- `like_count`
- `comment_count`
- `created_at`
- `updated_at`

### 4.4 设计说明

- `slug` 应唯一
- `status` 当前先简化为 `DRAFT`、`PUBLISHED`、`OFFLINE`
- 正文内容不放在该表中
- 当前阶段文章直接关联 `user.id`，字段命名使用 `user_id`
- 统计字段直接放在主表，便于列表与详情快速展示
- `deleted` 用于软删除

### 4.5 当前阶段为什么这样设计

- 列表页、首页、推荐位、后台文章列表都会频繁读取 `post`
- 正文、HTML、目录、富内容块都比较大，不适合放在主表
- `post` 表应尽量保持轻量，便于查询和索引优化
- 详情页正文单独放到 `post_content` 表更合理

## 五、文章内容表 `post_content`

### 5.1 作用

存放当前生效正文内容与派生字段。

### 5.2 当前确认版 DDL

```sql
CREATE TABLE `post_content` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '内容ID',
  `post_id` BIGINT UNSIGNED NOT NULL COMMENT '文章ID',
  `content_markdown` LONGTEXT NOT NULL COMMENT 'Markdown正文',
  `content_html` LONGTEXT NOT NULL COMMENT 'HTML正文',
  `content_plain_text` LONGTEXT DEFAULT NULL COMMENT '纯文本内容',
  `content_toc_json` JSON DEFAULT NULL COMMENT '目录JSON',
  `video_url` VARCHAR(500) DEFAULT NULL COMMENT '视频直链地址',
  `reading_time` INT NOT NULL DEFAULT 0 COMMENT '预计阅读时长，单位分钟',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_post_content_post_id` (`post_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='文章内容表';
```

### 5.3 推荐字段

- `id`
- `post_id`
- `content_markdown`
- `content_html`
- `content_plain_text`
- `content_toc_json`
- `video_url`
- `reading_time`
- `created_at`
- `updated_at`

### 5.4 设计说明

- `content_markdown` 是主内容源
- `content_html` 是详情页展示加速字段
- `content_plain_text` 用于摘要提取、搜索预留、审核辅助
- `content_toc_json` 用于前端目录展示
- `video_url` 当前只支持单视频直链
- `post_id` 使用唯一索引，保证一篇文章当前只有一份生效内容

### 5.5 当前阶段为什么这样设计

- 文章详情页只需要读取当前生效内容，因此 `post_content` 与 `post` 一对一
- `content_markdown` 用于后续继续编辑
- `content_html` 用于详情页快速展示
- 当前阶段视频先采用单地址方案，不单独拆视频表
- 当前阶段先不引入富内容块字段，避免内容模型过早复杂化
- 如果后续需要历史版本，再使用 `post_content_version` 承载，而不是把多版本塞进当前表中

## 八、评论表 `comment`

### 8.1 作用

用于存放文章评论与评论回复，当前阶段仅支持文章评论。

### 8.2 当前确认版 DDL

```sql
CREATE TABLE `comment` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '评论ID',
  `post_id` BIGINT UNSIGNED NOT NULL COMMENT '文章ID',
  `user_id` BIGINT UNSIGNED NOT NULL COMMENT '评论用户ID',
  `reply_comment_id` BIGINT UNSIGNED DEFAULT NULL COMMENT '直接回复的评论ID',
  `root_id` BIGINT UNSIGNED DEFAULT NULL COMMENT '根评论ID',
  `content` TEXT NOT NULL COMMENT '评论内容',
  `status` VARCHAR(32) NOT NULL DEFAULT 'PENDING' COMMENT '状态：PENDING / APPROVED / REJECTED',
  `ip` VARCHAR(64) DEFAULT NULL COMMENT '评论IP',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否删除：0否 1是',
  PRIMARY KEY (`id`),
  KEY `idx_comment_post_id` (`post_id`),
  KEY `idx_comment_user_id` (`user_id`),
  KEY `idx_comment_reply_comment_id` (`reply_comment_id`),
  KEY `idx_comment_root_id` (`root_id`),
  KEY `idx_comment_status` (`status`),
  KEY `idx_comment_created_at` (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='评论表';
```

### 8.3 推荐字段

- `id`
- `post_id`
- `user_id`
- `reply_comment_id`
- `root_id`
- `content`
- `status`
- `ip`
- `created_at`
- `updated_at`
- `deleted`

### 8.4 设计说明

- `post_id` 关联文章
- `user_id` 关联用户
- `reply_comment_id` 用于标记当前评论直接回复的是哪条评论
- `root_id` 用于标记整个评论线程的根评论
- `status` 当前先使用 `PENDING`、`APPROVED`、`REJECTED`
- `deleted` 用于软删除

### 8.5 当前阶段为什么这样设计

- 支持普通评论与楼中回复
- 支持评论审核，适合个人内容站防垃圾评论
- 结构简单，后续可继续扩展点赞、置顶、管理员回复等能力

## 十二、站点配置表 `site_config`

### 12.1 作用

用于存放站点级配置，适合开源后由不同站长自定义站点名称、图标、SEO、主题和功能开关。

### 12.2 当前确认版 DDL

```sql
CREATE TABLE `site_config` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '配置ID',
  `config_key` VARCHAR(64) NOT NULL COMMENT '配置键，如 site.base / site.seo / site.theme / site.feature',
  `config_name` VARCHAR(100) NOT NULL COMMENT '配置名称',
  `config_json` JSON NOT NULL COMMENT '配置内容JSON',
  `remark` VARCHAR(255) DEFAULT NULL COMMENT '备注',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_site_config_config_key` (`config_key`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='站点配置表';
```

### 12.3 推荐字段

- `id`
- `config_key`
- `config_name`
- `config_json`
- `remark`
- `created_at`
- `updated_at`

### 12.4 设计说明

- 当前阶段使用单表即可，不拆多张配置表
- 逻辑上按配置分组，表上通过 `config_key` 区分
- `config_json` 适合存放站点级结构化配置
- `config_key` 当前建议至少包含：
  - `site.base`
  - `site.seo`
  - `site.theme`
  - `site.feature`

### 12.5 示例说明

#### `site.base`

适合存放：

- 网站名称
- 副标题
- Logo
- favicon
- 页脚信息

#### `site.seo`

适合存放：

- 默认 SEO 标题
- 默认 SEO 描述
- 默认分享图

#### `site.theme`

适合存放：

- 主色
- 辅助色
- 圆角风格
- 首页样式

#### `site.feature`

适合存放：

- 是否允许注册
