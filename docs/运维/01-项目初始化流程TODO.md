# 项目初始化流程 TODO

## 文档目标

本文档用于定义 `云屿 / Yunyu` 第一阶段项目初始化流程待办。

重点不是实现代码，而是明确后续系统首次启动时需要完成哪些启动检查、自动建库建表动作，避免后端开发时遗漏关键流程。

## 一、初始化触发时机

当前建议初始化流程只在以下场景触发：

- 项目第一次启动
- 数据库为空，核心表尚未创建
- 系统检测到尚未完成“站点初始化”

不建议在每次正常启动时都进入交互式初始化流程。

系统后续应满足：

- 第二次及之后启动时自动判断是否已经初始化完成
- 如果状态为 `READY`，则直接进入正常运行态
- 已初始化时自动跳过数据库与表结构初始化逻辑

## 二、初始化目标

系统首次启动时，至少要完成以下目标：

- 检查 `spring.datasource` 数据库配置是否存在
- 检查数据库是否可连接
- 检查目标数据库是否存在
- 如果不存在则自动创建数据库
- 检查核心表是否已经创建
- 如未创建，则自动执行初始化 SQL
- 标记系统初始化完成

## 四、建议的状态判断规则

当前建议将系统状态分为三种：

- `UNINITIALIZED`
  数据库不可用，或核心表不存在
- `PARTIALLY_INITIALIZED`
  表已创建，但系统尚未达到完整可用状态
- `READY`
  数据库、站点配置、站长账号均已完成初始化
