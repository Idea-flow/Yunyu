# 项目初始化流程 TODO

## 文档目标

本文档用于定义 `云屿 / Yunyu` 第一阶段项目初始化流程待办。

重点不是实现代码，而是明确后续系统首次启动时需要完成哪些初始化检查、交互和落库动作，避免后端开发时遗漏关键流程。

## 一、初始化触发时机

当前建议初始化流程只在以下场景触发：

- 项目第一次启动
- 数据库为空，核心表尚未创建
- 系统检测到尚未完成“站点初始化”

不建议在每次正常启动时都进入交互式初始化流程。

## 二、初始化目标

系统首次启动时，至少要完成以下目标：

- 检查数据库配置是否存在
- 检查数据库是否可连接
- 检查核心表是否已经创建
- 如未创建，则执行初始化 SQL
- 检查站点基础配置是否完整
- 检查是否已存在站长账号
- 如不存在，则引导创建首个 `SUPER_ADMIN`
- 标记系统初始化完成

### 3.1 启动阶段检测

### 3.3 站点基础信息初始化

- [ ] 检查 `site_config` 中是否存在 `site.base`
- [ ] 检查站点名称 `siteName` 是否为空
- [ ] 如果为空，进入“站点信息初始化”流程
- [ ] 需要用户输入站点名称

- [ ] 可选输入默认 SEO 标题与描述

当前阶段建议至少采集：

- 站点名称
- 站点副标题
- 默认 SEO 标题
- 默认 SEO 描述

### 3.4 站长账号初始化

- [ ] 检查 `user` 表中是否存在 `role = 'SUPER_ADMIN' and deleted = 0` 的账号
- [ ] 如果不存在，则进入“站长账号创建”流程
- [ ] 需要输入站长邮箱
- [ ] 需要输入站长展示名
- [ ] 需要输入初始密码
- [ ] 写入 `password`
- [ ] 同时生成并写入 `password_hash`
- [ ] 角色设置为 `SUPER_ADMIN`
- [ ] 状态设置为 `ACTIVE`

### 3.5 初始化完成标记

- [ ] 初始化完成后写入系统标记
- [ ] 当前建议使用 `site_config` 增加一个 `system.init` 配置项
- [ ] 至少记录初始化时间、初始化版本、初始化人邮箱

## 四、建议的状态判断规则

当前建议将系统状态分为三种：

- `UNINITIALIZED`
  数据库不可用，或核心表不存在
- `PARTIALLY_INITIALIZED`
  表已创建，但站点信息或站长账号不完整
- `READY`
  数据库、站点配置、站长账号均已完成初始化
