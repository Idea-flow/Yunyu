# 后台接口故障排查

本文档用于指导 `yunyu-admin-operator` skill 在后台接口调用失败时，快速判断问题更可能出在 token、域名、权限、接口路径还是服务端本身。

## 一、统一排查顺序

当任一后台接口失败时，建议固定按以下顺序排查：

1. 先确认本地连接信息是否存在且与当前环境一致。
2. 再确认当前 token 是否有效。
3. 再确认当前 token 是否对应 `SUPER_ADMIN`。
4. 再确认请求路径、方法、参数是否与 `admin-api-docs.md` 一致。
5. 最后再判断是否属于服务端异常或目标环境不可用。

推荐先执行：

```bash
bash .agents/skills/yunyu-admin-operator/scripts/admin_connection.sh show
```

如需切换连接信息：

```bash
bash .agents/skills/yunyu-admin-operator/scripts/admin_connection.sh set --base-url <URL> --token <TOKEN>
```

## 二、401 Unauthorized

通常表示：

- token 缺失
- token 过期
- token 无效
- 请求头里的 Bearer token 格式错误

优先判断：

1. 本地连接文件里的 token 是否为空或明显不对。
2. 用户是否切换了环境，但还在使用旧 token。
3. token 是否仍然对应已登录后台会话。

建议动作：

1. 先让用户确认当前后台域名是否正确。
2. 再让用户重新提供 token，或重新从浏览器执行：
   `localStorage.getItem('yunyu_access_token')`
3. 保存新的连接信息后重试。

不要优先假设是业务字段错误。

## 三、403 Forbidden

通常表示：

- token 有效，但当前用户权限不足
- 当前用户不是 `SUPER_ADMIN`
- 某些敏感接口需要更高权限策略

优先判断：

1. 是否已经先调用 `/api/auth/me` 校验当前用户身份。
2. 当前用户角色是否为 `SUPER_ADMIN`。
3. 是否误用了前台普通用户 token。

建议动作：

1. 先提示用户当前 token 可能有效，但权限不足。
2. 让用户更换为站长后台 token。
3. 如有必要，重新调用 `/api/auth/me` 做确认。

## 四、404 Not Found

通常有两种可能：

- 域名或基础路径错误
- 接口路径或资源 ID 错误

优先判断：

1. `baseUrl` 是否指向了正确的后台环境。
2. 当前环境是否真的部署了对应接口版本。
3. 请求路径是否和 `references/admin-api-docs.md` 完全一致。
4. 如果是详情、更新、删除接口，资源 ID 是否存在。

判断技巧：

1. 如果所有接口都 404，更可能是 `baseUrl` 错了。
2. 如果只有某个资源详情 404，更可能是资源 ID 不存在。
3. 如果是单个新接口 404，也可能是 skill 文档落后于当前部署环境。

建议动作：

1. 先确认域名和环境。
2. 再确认接口路径和方法。
3. 最后确认目标资源是否真实存在。

## 五、500 / 502 / 503 / 504

通常表示：

- 服务端内部异常
- 目标环境未启动或不稳定
- 反向代理、网关、超时问题

优先判断：

1. 是否只有当前接口失败，还是整个后台都不可用。
2. 是否刚好涉及 AI 提供商、S3、外部服务测试等更容易超时的接口。
3. 是否是目标环境本身网络不通、网关异常或服务未启动。

建议动作：

1. 先告诉用户这更像服务端或环境问题，不要马上重复提交写操作。
2. 如果是写操作，避免在结果不明确时连续重试，防止重复写入。
3. 如用户愿意，可先改用一个简单读接口验证环境是否整体可用。

## 六、如何区分“token 问题”与“域名问题”

更像 token 问题的信号：

- 401
- `/api/auth/me` 失败
- 同一域名下所有后台接口都提示未认证

更像域名问题的信号：

- 404 且多个接口都 404
- 连接超时、DNS 错误、网关错误
- 用户明确说自己在另一个部署环境操作

更像权限问题的信号：

- 403
- `/api/auth/me` 能返回用户信息，但角色不是 `SUPER_ADMIN`

## 七、推荐对用户的反馈方式

当失败时，建议直接告诉用户：

1. 失败的接口和状态码
2. 当前更怀疑是哪一类问题
3. 下一步希望用户提供什么

推荐表达：

- `当前请求返回 401，更像是 token 失效或环境不匹配。要不要更换当前后台 token 后重试？`
- `当前请求返回 403，说明 token 可能有效，但当前账号不是 SUPER_ADMIN。要不要更换站长 token？`
- `当前请求返回 404，更像是当前 baseUrl 不对，或者这个环境没有对应接口。要不要确认一下后台域名？`
- `当前请求返回 5xx，更像服务端或环境异常。我建议先不要重复提交写操作，要不要先换一个简单查询接口确认环境状态？`
